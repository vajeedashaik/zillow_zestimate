PROJECT CONTEXT
══════════════════════════════════════════

WHAT THIS PROJECT DOES:
Predicts logerror = log(Zestimate) - log(SalePrice) for 2.9M California properties.
- Positive logerror → Zestimate overestimated the home value (seller beware)
- Negative logerror → Zestimate underestimated (buyer opportunity)
- logerror ≈ 0 → Zestimate was accurate

DATASET:
- properties_2016.csv: ~2.9M rows, LA (FIPS 6037), Orange (FIPS 6059), Ventura (FIPS 6111) counties
- train_2016.csv: ~90K transactions with logerror labels
- Evaluation metric: RMSE (lower is better)

MODEL RESULTS (CV RMSE):
| Model          | RMSE   |
|----------------|--------|
| Ridge          | 0.0791 |
| ExtraTrees     | 0.0763 |
| XGBoost        | 0.0754 |
| CatBoost       | 0.0751 |
| LightGBM       | 0.0749 |
| Ensemble Stack | 0.0742 | ← best

PIPELINE STEPS (Model.py):
1. Data loading + dtype optimisation
2. EDA + outlier clipping (p1/p99)
3. Missing value handling (drop >90% sparse, median/mode imputation)
4. Feature engineering (12 new features)
5. Spatial features: KMeans k=25 on lat/lon, cluster density
6. Cross-validated target encoding (5-fold, no leakage)
7. Multicollinearity reduction (corr >0.95, VIF >20)
8. Baseline models: Ridge + ExtraTrees
9. Boosting: XGBoost, LightGBM, CatBoost (5-fold CV)
10. Bayesian tuning: Optuna (20 trials) on LightGBM
11. OOF Stacking: base model predictions → Ridge meta-model
12. SHAP explainability: TreeExplainer on XGBoost

ENGINEERED FEATURES:
- property_age = 2016 - yearbuilt
- tax_ratio = taxamount / (taxvaluedollarcnt + 1)
- structure_land_ratio = structureTax / (landTax + 1)
- living_area_ratio = finishedSqFt / (lotSizeSqFt + 1)
- month_sin / month_cos (cyclical encoding)
- geo_cluster (KMeans k=25 on normalised lat/lon)
- geo_cluster_density (property count per cluster)
- cluster_mean_tax
- geo_cluster_te (CV mean logerror per cluster)
- county_te (CV mean logerror per county)
- age_cluster_interaction = property_age × geo_cluster

Client-side feature engineering (lib/featureEngineering.ts):
- property_age = 2016 - yearBuilt
- tax_ratio = taxAmount / (taxValue + 1)
- living_area_ratio = finishedSqFt / (lotSize + 1)
- month_sin = Math.sin(2 * Math.PI * month / 12)
- month_cos = Math.cos(2 * Math.PI * month / 12)
- county_te: LA=0.0142, Orange=-0.0089, Ventura=0.0031
- geo_cluster_te: derive from tax_ratio buckets (low<0.01: -0.008, mid 0.01-0.02: 0.012, high>0.02: 0.024)

Prediction formula (lib/mockPredictor.ts):
logerror = 0.045 + (tax_ratio * 1.82) + (county_te * 0.8) + (geo_cluster_te * 0.6)
           - (property_age * 0.00015) + (living_area_ratio * 0.003)
           + small random noise (±0.002 seeded by inputs for consistency)
confidence_low = logerror - 0.018
confidence_high = logerror + 0.018

Results to display:
- Predicted logerror value (large, styled number)
- Estimated true value range:
  true_low = zestimate / Math.exp(confidence_high)
  true_high = zestimate / Math.exp(confidence_low)
